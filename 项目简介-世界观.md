# 世界观 — AI 多世界科普平台

> 每个学科都有你看不懂的名词。但如果让诸葛亮在三国战场上给你讲"博弈论"、让武林高手在华山之巅演示"熵增"——你一辈子都忘不掉。

---

## 一、产品定义

### 一句话

**用你熟悉的世界，解释陌生的概念。** 选择一个世界观，输入一个你看不懂的名词，AI 在那个世界里用一个寓言故事把它讲明白——全程不出现术语，等你恍然大悟后再告诉你"刚才你理解的概念叫这个"。

### 核心学习闭环

```
选世界 → 输入概念 → AI 讲故事（不出现术语）
  → 揭晓概念 + 故事到概念的逐一映射
  → 两个检验问题（理解检验 + 迁移检验）
  → 完成一次真正的理解
```

### 和 ChatGPT 的差异

| | 直接问 ChatGPT | 世界观 |
|------|------|------|
| 解释方式 | 教科书式定义 + 举例 | 沉浸式故事包裹概念 |
| 体验感 | 读完可能忘 | "先理解，后知道在学什么" |
| 质量 | 取决于你问得好不好 | 每个世界 prompt 调优过，质量稳定 |
| 复购 | 无 | 同一个概念在 3 个世界看 3 种解释 |

## 二、用户路径

### 主流程

```
首页（世界选择器）
  → 5 个世界卡片，每个有独特视觉主题
  → 用户选一个世界 → 进入科普页

科普页
  → 输入框 + "生成寓言"按钮
  → 生成后展示：
      1. 寓言正文（沉浸式阅读）
      2. 概念解析（折叠面板）
      3. 检验问题（折叠面板）
  → 底部："换个世界看同一个概念"按钮

Demo 最佳体验路径：
  评委说一个词（如"内卷"）
  → 零点便利店世界生成寓言 → 评委读完
  → 点"换个世界" → 三国世界生成同一个概念的解释
  → "原来内卷的本质是……" ← 跨世界理解发生
```

### 页面清单（3 个页面）

| 页面 | 路由 | 内容 |
|------|------|------|
| 首页 | `/` | 世界选择器（卡片网格） + 历史记录入口 |
| 科普页 | `/explore/:worldId` | 输入概念 → 生成寓言 → 查看解析 |
| 历史页 | `/history` | 所有生成过的寓言列表 |

### 5.2 世界数据（src/data/worlds.ts）

**重要**：这个文件用静态 import 导入每个世界的 `world.json`。TRAE IDE 在创建项目时需要先创建好 `worlds/` 目录下的 `world.json` 文件，然后在 `vite.config.ts` 中配置 alias 以便用 `@worlds/` 前缀导入。

建议简化方案：**不在构建时 import，而是在运行时用 fetch 读取**。这样不需要处理 Vite 的静态资源导入配置。

```typescript
import type { WorldMeta } from '@/types';

// 世界 ID 列表
export const WORLD_IDS = [
  'midnight-store',
  'three-kingdoms',
  'martial-arts',
  'kitchen',
  'codebase',
] as const;

// 缓存
let worldsCache: WorldMeta[] | null = null;

/** 加载所有世界元信息（运行时 fetch world.json） */
export async function loadWorlds(): Promise<WorldMeta[]> {
  if (worldsCache) return worldsCache;
  
  const worlds = await Promise.all(
    WORLD_IDS.map(async (id) => {
      const res = await fetch(`/worlds/${id}/world.json`);
      const data = await res.json();
      return { ...data, id } as WorldMeta;
    })
  );
  
  worldsCache = worlds;
  return worlds;
}

/** 获取单个世界 */
export async function getWorld(worldId: string): Promise<WorldMeta | undefined> {
  const worlds = await loadWorlds();
  return worlds.find(w => w.id === worldId);
}
```

```typescript
const STORAGE_KEY = 'worldview_llm_config';

/** 默认配置 */
const DEFAULT_CONFIG: LLMConfig = {

  apiKey: '',

  { label: 'DeepSeek Chat', model: 'deepseek-chat', baseURL: 'https://api.deepseek.com/v1', apiKey: '' },
  { label: '通义千问 Plus', model: 'qwen-plus', baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1', apiKey: '' },
  { label: 'vLLM 本地部署', model: 'Qwen3.6-27B', baseURL: 'http://192.168.1.119:8012/v1', apiKey: 'EMPTY' },
];

/** 读取本地存储的配置 */
export function loadLLMConfig(): LLMConfig {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    if (raw) return { ...DEFAULT_CONFIG, ...JSON.parse(raw) };
  } catch { /* ignore */ }
  return DEFAULT_CONFIG;
}

/** 保存配置 */
export function saveLLMConfig(config: LLMConfig): void {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(config));
}

/** 是否为本地部署 */
export function isLocalDeployment(config: LLMConfig): boolean {
  const url = config.baseURL.toLowerCase();
  return url.includes('localhost') || url.includes('127.0.0.1') || url.includes('192.168.') || url.includes('10.');
}

/** 检查 LLM 是否可用 */
export function isLLMAvailable(config: LLMConfig): boolean {
  if (!config.baseURL || !config.model) return false;
  if (isLocalDeployment(config)) return true;
  return !!config.apiKey;
}

/** Qwen3 <think> 标签清理 */
function cleanThinkTags(text: string): string {
  return text
    .replace(/<think>[\s\S]*?<\/think>/gi, '')
    .replace(/<think>[\s\S]*$/gi, '')
    .trim();
}

/**
 * 调用 OpenAI 兼容接口
 */
export async function chatCompletion(
  config: LLMConfig,
  systemPrompt: string,
  userPrompt: string,
  options?: { maxTokens?: number; temperature?: number; timeoutMs?: number }
): Promise<string> {
  const url = `${config.baseURL.replace(/\/$/, '')}/chat/completions`;

  const body = {
    model: config.model,
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: userPrompt },
    ],
    max_tokens: options?.maxTokens ?? 2000,
    temperature: options?.temperature ?? 0.7,
  };

  const headers: Record<string, string> = { 'Content-Type': 'application/json' };
  if (config.apiKey) {
    headers['Authorization'] = `Bearer ${config.apiKey}`;
  }

  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), options?.timeoutMs ?? 60000);

  try {
    const response = await fetch(url, {
      method: 'POST',
      headers,
      body: JSON.stringify(body),
      signal: controller.signal,
    });

    if (!response.ok) {
      const errorText = await response.text();
      throw new Error(`LLM 请求失败 (${response.status}): ${errorText}`);
    }

    const data = await response.json();
    const content = data.choices?.[0]?.message?.content ?? '';
    return cleanThinkTags(content);
  } finally {
    clearTimeout(timeoutId);
  }
}

/** 测试连接 */
export async function testConnection(config: LLMConfig): Promise<boolean> {
  try {
    const result = await chatCompletion(
      config,
      '你是一个测试助手。',
      '请回复"连接成功"四个字。',
      { maxTokens: 20, temperature: 0, timeoutMs: 15000 }
    );
    return result.length > 0;
  } catch {
    return false;
  }
}
```

### 5.4 Prompt 组装（src/utils/promptBuilder.ts）

```typescript
/**
 * 加载世界 system prompt + 组装完整的 LLM 请求参数
 */
export async function buildFablePrompt(
  worldId: string,
  concept: string
): Promise<{ systemPrompt: string; userPrompt: string }> {
  // 运行时 fetch 世界的 system-prompt.md
  const response = await fetch(`/worlds/${worldId}/system-prompt.md`);
  let systemPrompt = await response.text();
  
  // 替换占位符（如果 system-prompt.md 中有 {concept} 占位符）
  systemPrompt = systemPrompt.replace(/\{concept\}/g, concept);
  
  // user prompt 简单直接
  const userPrompt = `请围绕"${concept}"这个概念，写一则寓言。`;
  
  return { systemPrompt, userPrompt };
}
```

### 5.5 响应解析（src/utils/parseResponse.ts）

LLM 输出的格式固定为：

```
[寓言正文，无标题]

---

**概念解析**
1. 概念：[名称]，属于[学科/流派]，核心定义是……
2. 故事映射：
   - [故事元素A] → [概念部分A]
   - [故事元素B] → [概念部分B]

---

**检验问题**
1. [理解检验问题]
2. [迁移检验问题]
```

解析器实现：

```typescript
import type { FableMapping } from '@/types';

interface ParsedFable {
  content: string;
  conceptName: string;
  conceptDefinition: string;
  mappings: FableMapping[];
  understandingQuestion: string;
  transferQuestion: string;
}

export function parseFableResponse(raw: string): ParsedFable | null {
  // 按 --- 分割三部分
  const parts = raw.split('\n---\n');
  if (parts.length < 3) {
    // 尝试宽松匹配（有些模型可能不用 --- 分隔）
    return null;
  }

  const content = parts[0].trim();
  const analysisSection = parts[1].trim();
  const quizSection = parts[2].trim();

  // 从解析部分提取概念名和定义
  const conceptMatch = analysisSection.match(
    /概念[：:]\s*(.+?)[，,]\s*属于(.+?)[，,]\s*核心定义[是为：:]\s*(.+?)(?:\n|$)/
  );
  const conceptName = conceptMatch?.[1]?.trim() ?? '';
  const conceptDefinition = conceptMatch?.[3]?.trim() ?? '';

  // 提取映射列表
  const mappings: FableMapping[] = [];
  const mappingRegex = /[-•]\s*(.+?)\s*[→>]\s*(.+?)(?:\n|$)/g;
  let match;
  while ((match = mappingRegex.exec(analysisSection)) !== null) {
    mappings.push({
      storyElement: match[1].trim(),
      conceptPart: match[2].trim(),
    });
  }

  // 提取两个问题
  const questionRegex = /\d+[.、]\s*(.+?)(?:\n\d+[.、]|\n*$)/gs;
  const questions: string[] = [];
  let qMatch;
  while ((qMatch = questionRegex.exec(quizSection)) !== null) {
    questions.push(qMatch[1].trim());
  }

  return {
    content,
    conceptName,
    conceptDefinition,
    mappings,
    understandingQuestion: questions[0] ?? '',
    transferQuestion: questions[1] ?? '',
  };
}
```

### 5.6 Zustand Store（src/store/useAppStore.ts）

```typescript
import { create } from 'zustand';
import type { AppState, LLMConfig, FableResult } from '@/types';
import { loadLLMConfig, saveLLMConfig, isLLMAvailable } from '@/services/llmService';

interface AppActions {
  setCurrentWorld: (worldId: string | null) => void;
  setCurrentConcept: (concept: string) => void;
  setLLMConfig: (config: LLMConfig) => void;
  setLLMSettingsOpen: (open: boolean) => void;
  setIsGenerating: (generating: boolean) => void;
  setError: (error: string | null) => void;
  addToHistory: (fable: FableResult) => void;
  clearHistory: () => void;
}

export const useAppStore = create<AppState & AppActions>((set, get) => ({
  // 初始状态
  currentWorldId: null,
  currentConcept: '',
  llmConfig: loadLLMConfig(),
  llmSettingsOpen: false,
  history: [],
  isGenerating: false,
  error: null,

  // Actions
  setCurrentWorld: (worldId) => set({ currentWorldId: worldId, currentConcept: '', error: null }),
  setCurrentConcept: (concept) => set({ currentConcept: concept }),
  setLLMConfig: (config) => {
    saveLLMConfig(config);
    set({ llmConfig: config });
  },
  setLLMSettingsOpen: (open) => set({ llmSettingsOpen: open }),
  setIsGenerating: (generating) => set({ isGenerating: generating }),
  setError: (error) => set({ error }),
  addToHistory: (fable) => set((state) => ({ history: [fable, ...state.history] })),
  clearHistory: () => set({ history: [] }),
}));
```

### system-prompt.md（核心！每个世界需要单独编写）

零点便利店的世界已有现成的完整设定（`tellme/fables/` 下的文件），直接合并即可。**其他 4 个世界需要手工编写 system-prompt.md。**

编写结构如下（以三国世界为例）：

```markdown
# 世界观：三国

## 基础设定
- **世界名称**：三国时代（公元 184-280 年）
- **世界观概述**：东汉末年至三国鼎立的历史时期。曹操、刘备、孙权三方势力逐鹿中原。这是一个谋略、博弈和权力交织的时代。
- **核心地理**：许昌（曹魏都城）、成都（蜀汉都城）、建业（东吴都城）、赤壁、官渡、长坂坡。
- **社会规则**：
  1. 军师出谋划策，武将冲锋陷阵
  2. 一场战役的胜负往往在开战前就由谋略决定了
  3. 同盟随时可能破裂，敌人随时可能结盟
  4. 每个人都在权衡利弊——没有永远的敌人，只有永远的利益

## 角色群
- **诸葛亮**：蜀汉丞相，字孔明。羽扇纶巾，谋略过人。喜欢用比喻和对比来讲道理。
- **曹操**：魏王，字孟德。枭雄，务实，说话直接甚至尖刻，但往往一针见血。
- **刘备**：蜀汉先主，字玄德。仁厚之名远播，擅长凝聚人心。
- **关羽**：蜀汉大将，字云长。忠义勇武，但不善于变通。
- **周瑜**：东吴都督，字公瑾。才华横溢但气量略小。
- **司马懿**：魏国谋臣，字仲达。极能忍，擅长持久战和消耗战。

## 寓言写作约束

- **篇幅**：1000 字以内
- **角色**：不超过 3 个，最好 2 个，从角色群中选择
- **故事场景**：必须发生在上面的核心地理范围内
- **揭示节奏**：全程不出现概念名称和领域术语，只在结尾让人隐约感知
- **叙事纪律**：情节和细节承载意义，角色不跳出来解说
- **语言风格**：半文半白。对话可带文言色彩但不晦涩。可使用"却说""且说""忽一日"等话本风格过渡。人物对话体现各自性格。
- **禁止套路**：不要写"某日某人在帐中沉思""军师微微一笑说出玄机"这种烂俗桥段

## 输出格式

### 第一部分：寓言正文
直接输出故事，不加标题、不加引导语。

### 第二部分：概念解析
1. 概念：[名称]，属于[学科/流派]，核心定义是……
2. 故事映射：
   - [故事元素 A] → [概念的某一部分]
   - [故事元素 B] → [概念的某一部分]

### 第三部分：检验问题
1. [理解检验：检验是否真正理解了概念的核心]
2. [迁移检验：检验能否将概念迁移到其他领域]
```

## 七、页面组件说明

### Layout.tsx（全局布局）

- 顶部导航栏：左侧 Logo"世界观"，右侧 LLM 设置按钮（齿轮图标）
- `<Outlet />` 渲染子页面
- 底部："用你熟悉的世界，理解陌生的概念"

### HomePage.tsx（首页）

- 标题 + 副标题
- WorldSelector 组件：5 张世界卡片网格
- 每张卡片：emoji + 世界名 + 一句介绍 + 主题色边框
- 点击卡片进入 `/explore/:worldId`
- 底部：历史记录入口（"📖 我的学习记录"）

### ExplorePage.tsx（科普页 — 核心页面）

- URL 参数 `:worldId` 确定当前世界
- 顶部：世界名称 + tagline + "切换世界"下拉按钮
- 概念输入区：大号 input + "生成寓言"按钮 + 预设概念标签
- **生成前**：展示预设概念标签（内卷、熵增、边际效应、认知失调、奥德赛时期）
- **生成中**：loading 动画（"正在从这个世界理解这个概念……"）
- **生成后**：
  - 寓言正文区（深色文字衬线体，大幅留白）
  - 文末一个温和的分隔符（"✦"）
  - ConceptReveal 折叠面板（默认折叠）
  - QuizPanel 折叠面板（默认折叠）
- 底部操作栏："分享" + "换个世界看同一个概念"（醒目按钮）

### HistoryPage.tsx（历史记录）

- 按时间倒序展示所有生成过的寓言
- 每条：概念名 + 世界名 + 日期 + 预览前 50 字
- 点击展开完整内容或跳转到科普页

### LLMSettings.tsx（LLM 配置弹窗）

- 预设模型选择（下拉）
- baseURL / apiKey / model 输入框
- 连接测试按钮
- 保存 / 关闭

## 八、关键交互细节

1. **"换个世界看同一个概念"** — 这是 killer feature。用户读完一个世界的解释后，点击此按钮弹出世界选择器（排除当前世界），选择后自动用同一概念在新世界生成寓言
2. **概念解析默认折叠** — 用户必须先读完故事，手动展开才能看到概念名，保持"先理解、后知道"的体验
3. **打字机/逐段动画** — 寓言正文不是一次性出现，而是逐段淡入（利用 CSS animation-delay）
4. **LLM 设置** — 第一次使用时如果没有配置 API Key，弹出 LLMSettings 引导用户配置
5. **错误处理** — LLM 调用失败时显示友好提示："这个故事暂时讲不出来，换个世界试试？"
6. **移动端适配** — 最大阅读宽度 640px，中屏和手机均居中显示

## 十、报名与路演

### 赛道：学习工作（学习方式的创新）

### 报名帖标题

"同一个概念，在便利店、三国战场和武林江湖里，是完全不同的故事"

### 路演 2 分钟稿

> 我第一次看到"奥德赛时期"这个词的时候，完全不知道是什么意思。直到有人用外卖骑手阿浩的故事讲给我听——三分钟，一个故事，我一辈子忘不掉。
>
> 这就是世界观：不是让 AI 替你查资料，是让 AI 在你最熟悉的世界里，用一个故事把最陌生的概念讲明白。
>
> 你现在心里想一个词——任何一个你一直想问但懒得搜的概念。我们现场试试。

然后评委说一个词 → 当场生成 → 读完 → 评委的恍然大悟就是最好的 Demo。

### 为什么这个产品能赢

1. **概念新鲜**：33 个官方灵感里没有这个角度
2. **体验即时**：评委 30 秒从"听不懂"到"明白了"
3. **技术诚实**：纯 LLM + 前端，零外部依赖，没有技术债
4. **有壁垒**：5 个世界的 prompt 工程是踩坑调出来的，别人抄不了
5. **天然传播**：用户会截图发朋友圈——报名帖本身就是最好的展示
