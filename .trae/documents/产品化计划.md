# 世界观（Tellme）产品化计划

## 目标

将当前的纯前端原型升级为成熟的 C 端教育产品：加入后端服务、用户体系、数据持久化、API 安全代理、以及产品级体验优化。

***

## 一、当前状态分析

### 已有优势

* 核心功能完整：寓言生成（流式）、概念解析、检验测验、世界记忆（Agent 记忆机制）、多租户、历史记录

* 技术栈现代克制：React 18 + Vite 5 + Zustand + Tailwind

* 5 个内置世界，构建时自动发现，新增世界零代码改动

* 风格随机化 + 世界记忆连续性，产品差异化亮点

### 核心问题（阻碍产品化）

| 问题                        | 影响                   |
| ------------------------- | -------------------- |
| API Key 存浏览器 localStorage | 安全风险，用户 Key 可被窃取     |
| 无用户认证                     | 无法跨设备、无法收费、无法做社交     |
| localStorage 5-10MB 限制    | 数据量一大就溢出，历史和记忆丢失     |
| 无后端代理                     | LLM 调用直连，无法做限流/计费/审计 |
| 无测试                       | 改动无保障                |
| 无 CI/CD                   | 发布靠手动                |
| 无错误监控                     | 线上问题不可见              |
| 移动端未适配                    | C 端产品大量移动用户          |
| 无 SEO/PWA                 | 获客和留存受限              |

***

## 二、后端架构设计

### 技术选型

* **后端框架**：Python + FastAPI（异步、自带 OpenAPI 文档、LLM 生态丰富）

* **数据库**：PostgreSQL（Supabase 托管，自带 Auth）

* **认证**：Supabase Auth（邮箱/手机/微信 OAuth）

* **LLM 代理**：后端 API Gateway，用 httpx 异步调用 LLM 提供商，StreamingResponse 转发 SSE

* **部署**：Vercel（前端）+ Supabase（数据库/Auth）+ Railway/Fly.io（Python 后端）

### 后端 API 设计

```
/api/auth/
  POST   /register          注册
  POST   /login             登录
  POST   /logout            登出
  GET    /me                当前用户信息

/api/fables/
  POST   /generate          生成寓言（SSE 流式，代理 LLM 调用）
  GET    /                  列表（分页）
  GET    /:id               详情
  DELETE /:id               删除
  POST   /:id/share         生成分享链接

/api/quiz/
  POST   /judge             AI 评判（代理 LLM 调用）

/api/worlds/
  GET    /                  世界列表
  GET    /:id               世界详情
  PUT    /:id/enabled       启用/禁用世界

/api/memories/
  GET    /:worldId          获取世界记忆
  POST   /:worldId          添加世界记忆
  DELETE /:worldId          清空世界记忆

/api/user/
  PUT    /profile           更新资料
  PUT    /llm-config        保存用户自带 LLM 配置（加密存储）
  GET    /llm-config        获取用户 LLM 配置
  GET    /stats             学习统计
```

### 数据库表设计

```sql
-- 用户（Supabase Auth 已有 auth.users，此处为扩展信息）
CREATE TABLE public.profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  username TEXT,
  avatar TEXT DEFAULT '👤',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 寓言记录
CREATE TABLE public.fables (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users NOT NULL,
  world_id TEXT NOT NULL,
  concept TEXT NOT NULL,
  concept_name TEXT,
  concept_definition TEXT,
  content TEXT NOT NULL,        -- 寓言正文
  mappings JSONB DEFAULT '[]',  -- 故事→概念映射
  quiz JSONB DEFAULT '{}',      -- 检验问题
  memory_entry JSONB,           -- 世界记忆条目
  is_shared BOOLEAN DEFAULT FALSE,
  share_token TEXT UNIQUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 世界记忆
CREATE TABLE public.world_memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users NOT NULL,
  world_id TEXT NOT NULL,
  fable_id UUID REFERENCES public.fables,
  concept TEXT,
  concept_name TEXT,
  story_summary TEXT,
  characters_involved TEXT[],
  world_changes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 测验作答
CREATE TABLE public.quiz_answers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users NOT NULL,
  fable_id UUID REFERENCES public.fables NOT NULL,
  question_key TEXT NOT NULL,
  answer TEXT,
  feedback TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, fable_id, question_key)
);

-- 用户 LLM 配置（加密存储 API Key）
CREATE TABLE public.user_llm_configs (
  user_id UUID REFERENCES auth.users PRIMARY KEY,
  base_url TEXT,
  api_key_encrypted TEXT,       -- AES 加密
  model TEXT,
  use_platform_llm BOOLEAN DEFAULT TRUE,  -- 是否使用平台提供的 LLM
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 启用的世界
CREATE TABLE public.user_worlds (
  user_id UUID REFERENCES auth.users NOT NULL,
  world_id TEXT NOT NULL,
  enabled BOOLEAN DEFAULT TRUE,
  PRIMARY KEY (user_id, world_id)
);
```

## 四、假设与决策

| 决策点    | 选择                         | 理由                                |
| ------ | -------------------------- | --------------------------------- |
| 后端框架   | Python + FastAPI           | 异步原生、自带 OpenAPI 文档、LLM 生态丰富、开发速度快 |
| 数据库    | Supabase (PostgreSQL)      | 自带 Auth/RLS/存储，C 端产品首选            |
| 认证方式   | Supabase Auth（邮箱+OAuth）    | 降低自建成本，支持微信/Google                |
| LLM 代理 | 后端统一代理                     | 隐藏平台 API Key，支持限流计费               |
| 部署     | Vercel + Supabase          | 前端 Vercel 自动部署，数据库 Supabase 托管    |
| 数据迁移   | 首次登录自动迁移 localStorage → 后端 | 无感升级，不丢老用户数据                      |
| 分享方案   | 生成 share\_token，公开页可查看     | 无需登录即可看，利于传播                      |
| PWA    | Vite PWA 插件                | 离线可用，可安装到桌面                       |

## 六、验证方式

* 后端 API：用 curl/Postman 逐个接口测试

* 认证流程：注册→登录→生成寓言→登出→重新登录数据还在

* 数据迁移：老 localStorage 数据 → 登录后 → 后端有数据 → localStorage 清理

* LLM 代理：前端不出现 API Key，生成正常

* 移动端：Chrome DevTools 移动端模拟 + 真机测试

* 分享：生成分享链接 → 无痕窗口打开 → 可查看寓言

* CI/CD：PR 自动跑检查，merge 后自动部署
