# 实现计划：世界观 — AI 多世界科普平台

## 概述

在 `tellme/` 根目录从零创建一个完整的 React + Vite + TypeScript + Tailwind CSS 3 Web 应用。产品核心：用户选择一个"世界观"，输入一个看不懂的概念，AI 在那个世界里用一则寓言（不出现术语）把概念讲明白，再揭晓概念名 + 故事到概念的映射 + 两个检验问题。

**5 个世界观即"skills"**：通过 Vite 的 `import.meta.glob` 在构建时动态发现 `worlds/*/` 目录下的所有世界，不硬编码世界 ID。前端可启用/禁用/查看任意世界的设定，新增一个世界文件夹即自动出现在应用中。

## 关键架构决策

| 决策点 | 选择 | 理由 |
|------|------|------|
| 项目位置 | `tellme/` 根目录 | 用户指定"就在顶层" |
| 世界发现机制 | `import.meta.glob('/worlds/*/world.json')` + `('/worlds/*/system-prompt.md', {query:'?raw'})` 构建时动态发现 | 满足"前端能使用所有 skills"，新增世界文件夹即自动生效，无需硬编码 ID、无需运行时 fetch |
| 每个世界的文件数 | 2 个：`world.json`（元信息+主题色合并）+ `system-prompt.md` | 简介原方案的 theme.json 合并进 world.json，减少文件数；WorldMeta 类型本就含 theme 字段 |
| system-prompt 加载 | 构建时 `?raw` 打包，非运行时 fetch | 静态 SPA 无需 public/ 目录，避免 404 风险 |
| 零点便利店 system-prompt | 合并 `fables_world.md` + `fables_create.md` | 用户指定，简介第 81 行明确要求 |
| 世界可选可配置 | localStorage 存储 enabledWorlds；UI 提供启用/禁用 + 查看设定面板 | 满足"skills 可选可配置" |
| 示例寓言 | 零点便利店附带 `sample-fable.md`（《嗯》），ExplorePage 提供"看示例"入口 | 无需 API 即可演示产品价值 |
| LLM 配置 | localStorage 持久化，首次使用引导配置 | 沿用简介 5.3 方案 |
| 路由 | HashRouter | 简介指定（静态部署友好） |

#### 3.2 `src/data/worlds.ts`（动态发现 — 核心改动）
```ts
import type { WorldMeta, WorldFull } from '@/types';

// 构建时动态发现所有世界（新增 worlds/xxx/ 文件夹即自动生效）
const worldJsonModules = import.meta.glob('/worlds/*/world.json', {
  eager: true, import: 'default',
}) as Record<string, WorldMeta>;

const promptModules = import.meta.glob('/worlds/*/system-prompt.md', {
  eager: true, query: '?raw', import: 'default',
}) as Record<string, string>;

function idFromPath(path: string): string | undefined {
  return path.match(/\/worlds\/(.+?)\//)?.[1];
}

// 解析出所有世界元信息
export const ALL_WORLDS: WorldMeta[] = Object.entries(worldJsonModules)
  .map(([path, data]) => ({ ...(data as object), id: idFromPath(path)! }))
  .filter(w => w.id) as WorldMeta[];

export const WORLD_IDS = ALL_WORLDS.map(w => w.id);

export function getWorldMeta(worldId: string): WorldMeta | undefined {
  return ALL_WORLDS.find(w => w.id === worldId);
}

// 获取某世界的 system prompt（构建时已打包）
export function getSystemPrompt(worldId: string): string {
  const entry = Object.entries(promptModules).find(
    ([p]) => idFromPath(p) === worldId
  );
  return entry?.[1] ?? '';
}

// 加载示例寓言（仅 midnight-store 有）
const sampleModules = import.meta.glob('/worlds/*/sample-fable.md', {
  eager: true, query: '?raw', import: 'default',
}) as Record<string, string>;
export function getSampleFable(worldId: string): string | undefined {
  const entry = Object.entries(sampleModules).find(
    ([p]) => idFromPath(p) === worldId
  );
  return entry?.[1];
}
```

## 假设与决策

1. **不使用 web-dev skill**：用户提供了完整且具体的技术规格（Vite+React+TS+Tailwind3+精确代码参考），按规格直接实现，避免 skill 强加不同技术栈。
2. **theme.json 合并进 world.json**：简介 WorldMeta 类型本含 theme 字段，合并后每世界 2 文件更简洁，解决简介 5.2 仅 fetch world.json 却需 theme 的不一致。
3. **system-prompt 构建时打包（?raw）而非运行时 fetch**：静态 SPA 无后端，构建时打包避免 public/ 配置和 404；同时 `import.meta.glob` 实现真正的动态发现。
4. **history 与 enabledWorldIds 持久化到 localStorage**：刷新不丢失，符合"可配置"预期。
5. **_tooling/ 保留 skill 工具链**：不删除 fable_builder 与 fables skill 基础设施，移至 _tooling/ 备查，保持根目录整洁。
6. **示例寓言仅零点便利店有**：其余 4 世界无现成成品，sample-fable 为可选，getSampleFable 返回 undefined 时不显示入口。
7. **strict: false**：沿用简介 tsconfig 配置，降低 TS 学习门槛，匹配原型项目定位。
